Etiquetas HTML avanzado

Etiqueta <center>

La etiqueta <center> se utilizaba em versiones antiguas de HTML para centrar contenido dentro de una página web, tanto texto como imágenes o otros elementos.

Está obsoleta en HTML5 y se tendria que evitar su uso. En lugar de <center>, hoy en día se usa CSS para centrar elementos de manera más eficiente.

Ejemplo:

    <center>
     <h1>Bienvenidos a mi sitio web>/h1>
    </center>

    IMPORTANTE: No usar más esta etiqueta, ya que está obsoleta.

Etiqueta <div>

  • La etiqueta <div> (abreviatura de "division") se utiliza para crear contenedores de bloque en una página HTML. Es una etiqueta de nivel de bloque, la cual cosa significa que ocupa todo el ancho disponible i comienza en una línea nueva.
  • No tiene significado semántico en si misma; simplemente agrupa cotros elementos, como texto, imágenes o otras etiquetas y suele ser estilizada con CSS.
  • Es útil para organizar i diseñar diferentes secciones de una página web.
    <div>
    <h2>Sección 1</h2>
    <p>Esto es un párrafo en un div :).</p>
    </div>

Buenas prácticas para usar divs en HTML:
  • Uno de los errores más comunes es el uso excesivo de <div> para cualquier cosa, cuando existen etiquetas semánticas como <header>, <section>, <article>, <footer>, ....
  • Utilizar <div> para agrupar elementos que forman una unidad lógica o visual dentro de una página, pero siempre que no haya una etiqueta semántica más adecuada.

Etiqueta <span>

  • La etiqueta <span> es una etiqueta inline (en línea) que sirve para estilizar o manipular pequeñas porciones de texto o contenido dentro de una página. A diferencia de <div>, no ocupa todo el ancho disponible, si no que ocupa sólo el espacio necesario.
  • También es una etiqueta no semántica, usada principalmente para aplicar estilos o funcionalidades a un grupo específico de elementos dentro de un texto.

    • Ejemplo:

      <p>Esto es un
       <span style="color: red";>texto en rojo</span>.
      </p>

    Esto es un texto en rojo.

Etiqueta <hr>

En HTML5 el uso de <hr> no es solo decorativo. Tiene un valor semántico para indicar un acnvio de tema o ruptura en el flujo del documento, ayudando tanto a los motores de busqueda como a las herramientas de accesibilidad a entender mejor la lectura del contenido.

Ejemplo:

    <h2>Historia de la compañía</h2>
    <p>Nuestra empresa fue fundada en el año 2026...</p>
    <hr>
    <h2>Nuestros servicios</h2>
    <p>Ofrecemos una alta gama de servicios.</p>

Buenas prácticas para usar divs en HTML:
    • No abusar de la etiqueta <hr>: Utilizarla sólo cuando sea necesario para separar secciones importantes o marcar un canvio de contenido.
    • Estilizar las líneas con CSS: Incluso si <hr> genera una línea por defecto, utiliza CSS para personalizar el aspecto y hacer que se ajuste al diseño de tu sitio web.
    • Uso semántico: Recuerda que <hr> se puede usar para indicar un canvio de contenido o de tema dentro de un documento, lo que también mejora la accesibilidad.
    • Evitar el uso de <hr> solo con finalidad visual.: Si necesitas una línea solo para estilo, considera usar CSS en lugar de la etiqueta <hr> para mantener el documento limpio i semántico.

Etiqueta <template>

La etiqueta <tempalte> se utiliza para definir fragmentos de HTML reutilizables que no se muestran inmediatamente cuando la página se carga. El contenido dentro de un <template> no se añade al documento hasta que se manipula explícitamente con JavaScript, cosa que permite crear plantillas que se pueden clonar i insertar al DOM de manera dinámica.

Este elemento es ideal cuando quieres mantener estructuras HTML preparadas que podrás añadir al documento en tiempo real, sin que estas sean rendidas o visibles inicialmente. Esto proporciona mucha flexibilidad para aplicaciones dinámicas.

Ejemplo:

    <template id="miPlantilla">
    <div class="producto">
    <h2>Nombre del producto</h2>
    <p>Descripción del producto</p>
    </div>
    </template>

Buenas prácticas para usar divs en HTML:
    • Mantener las plantillas simples i modulares: Organizar nuestro código dentro de <template> de manera que podamos reutilizar fácilmente estas plantillas en diferentes secciones de la página.
    • Clona plantillas con JavaScript: La etiqueta no renderiza su contenido hasta que lo invocas con JavaScript, la cual cosa nos da control total sobre como se tiene que añadir al DOM.
    • Utiliza plantillas para generar contenido dinámicamente: Es muy útil en aplicaciones donde el contenido tiene que ser generado o actualizado en tiempo real, como listas, formularios o tarjetas de productos.
    • No incluyas contenido activo dentro del template: Los scripts o otros contenidos interactivos dentro del template no se ejecutarán hasta que el contenido sea añadido al DOM, así que asegurate de añadir los elementos interactivos una vez esté el documento.

Etiqueta <Iframes>

<iframe> (inline frame) es una etiqueta HTML que permite insertar una página HTML dentro de otra página web. Es una manera más moderna i segura de incrustar contenido externo, como otras páginas web, vídoes, formularios o mapas, dentro de la página principal.

Los iframes se utilizan comunmente para contenidos dinámicos como vídeos de YouTube, Google Maps, o integraciones de servicios externos.

La etiqueta iframe se define dentro del cuerpo del documento i se puede personalizar con atributos como width, height, src i frameborder.

Ejemplo:

    <iframe width="560" height="315" src="https://www.youtube-nocookie.com/embed/xvFZjo5PgG0?si=LlGrng6iIS8IkwuH&controls=0" title="YouTube video player" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerpolicy="strict-origin-when-cross-origin" allowfullscreen></iframe>

Buenas prácticas para usar divs en HTML:
    • Evita el uso excesivo de iframes: El uso excesivo de iframes puede hacer que la página sea más lenta i más difícil de gestionar. Usalos sólo cuando sea necesario (como para incrustar contenido externo).
    • Utiliza el atributo sandbox por seguridad: Cuando incrustes contenido de otros sitios, utiliza el atributo sandbox para limitar las acciones que el iframe puede realizar dentro de tu sitio web, mejorando la seguridad.
    • Optimiza la altura y ancho: Ajusta la medida del iframe para asegurarte de que el contenido se vea correctamente en todas las medidas de pantalla, especialmente en dipositivos móviles.
    • Utiliza alternativas cuando sea posible: Cuando sólo necesites cargar contenido dinámico que puedas generat tu mismo (como un vídeo o una imagen), considera cargarlo directamente a la página en lugar de usar un iframe
    • Utiliza allowfullscreen para vídeos: Si estás incrustando vídeos, permite que se puedan visualizar en pantalla completa con el atributo allowfullscreen para mejorar la experiencia del usuario.
